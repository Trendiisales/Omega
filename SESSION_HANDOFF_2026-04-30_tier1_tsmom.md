## What this session delivered

### 1. `include/TsmomEngine.hpp` -- new self-contained header

Modelled on `C1RetunedPortfolio.hpp` (closest in-tree analog: same Phase 2 ->
C++ port pattern, same bar-based long-cell shape).

Public types (all in `namespace omega`):

| Type | Role |
|---|---|
| `TsmomBar`              | Minimal OHLC struct (mirror of `C1Bar`) |
| `TsmomBarSynth`         | H1 -> H2/H4/H6/D1 stride aggregator |
| `TsmomATR14`            | Wilder ATR14 (SMA bootstrap, then Wilder) |
| `TsmomCell`             | Single tsmom cell, parameterised by TF + direction |
| `TsmomPortfolio`        | Owns 5 long cells; H1 dispatch synthesises the rest |
| `TsmomPortfolio::Status`| Diagnostic accessor for monitors / dashboards |

Signal/exit logic mirrors `phase1/signal_discovery/post_cut_revalidate_all.py`
exactly:
- `sig_tsmom`: `ret_n = close[t] - close[t - 20]`, fire long if `ret_n > 0`
- `sim_c`: entry at next bar open; hard SL = `3.0 * ATR14_at_signal`;
  time exit at `hold_bars=12`; no TP

Anti-stacking: `cooldown_bars = hold_bars = 12` (see §"Caveat" below for why).

Macro regime: `block_on_risk_off=true` -- new entries blocked when
`g_macroDetector.regime() == "RISK_OFF"`. Existing positions still managed
by `on_tick` (intrabar SL fills) and `on_bar` (time exits).

Risk sizing: `size_for(sl_pts) = (equity * 0.005) / (sl_pts * 100)`,
floored 0.01, capped `max_lot_cap=0.05`. Exact mirror of
`C1RetunedPortfolio::size_for`, so shadow ledger PnL scales identically
across both portfolios.

### 2. Historical-bar warmup -- no cold-start window

`TsmomPortfolio::warmup_from_csv(path)` pre-warms `closes_` deques + ATRs +
synthesisers from a historical H1 bar CSV BEFORE live ticks arrive. Feeds
each historical bar through the same path live bars use, with `size_lot=0`
and a no-op `on_close` callback so warmup never produces `TradeRecord`s
or perturbs `equity_` / `peak_equity_` / `max_dd_pct_`.

CSV format (one row per H1 bar, comma-separated):
```
bar_start_ms,open,high,low,close
```
- Optional header (auto-skipped if first field is not a valid integer)
- Blank lines and `#`-prefixed lines are skipped
- Bars must be in ascending `bar_start_ms` order (parser does not re-sort)

CSV is generated by `phase1/signal_discovery/export_tsmom_warmup.py` from
`phase1/signal_discovery/bars_H1.parquet`. Already run this session ->
**6,156 H1 bars covering 2025-04-01 -> 2026-04-01** sit at
`phase1/signal_discovery/tsmom_warmup_H1.csv` (357 KB).

Cell readiness after warmup:

| Cell | closes need | will have | ATR14 status |
|---|---|---|---|
| H1 | 21      | 6,156    | ready |
| H2 | 21 H2   | 3,078 H2 | ready |
| H4 | 21 H4   | 1,539 H4 | ready |
| H6 | 21 H6   | 1,026 H6 | ready |
| D1 | 21 D1   | 256 D1   | ready |

So the very first live H1 bar can fire on every cell -- no 21-day D1 wait,
no 5-day H6 wait.

### 4. Caveat (carried from previous session, now addressed)

The simple `ret_n > 0` signal in `post_cut_revalidate_all.py` fires ~4x
more often than the master_summary canonical pipeline (3,484 H1 trades/yr
vs 807/2yr). Without a cooldown, deploying live would over-fire and hit
`max_positions` / `min_entry_gap` throttles, masking actual edge.

**Resolution shipped this session:** `cooldown_bars = hold_bars = 12`.
Per-trade edge (WR / PF / avg) is robust across the 4x frequency
difference, so this brings live behaviour into the master_summary regime
without losing the per-trade edge.

If first-week shadow ledger shows fire frequency still off the backtest
(>3x), revisit by re-deriving the canonical signal from
`phase1/sim_lib.py::TickReader` instead of relaxing the cooldown.
