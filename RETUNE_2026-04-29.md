## Runnable: post-Apr-2025 sweep script

Save as `backtest/run_post_regime_sweep.sh` (provided alongside this doc):

```bash
#!/bin/bash
# Re-run sweep on post-Apr-2025 portion of tick corpus only.
# Existing harness reads the full corpus -- we filter via awk first.
set -euo pipefail

REPO=/Users/jo/omega_repo
TICK_FULL=/Users/jo/Tick/duka_ticks/XAUUSD_2024-03_2026-04_combined.csv
TICK_POST=/Users/jo/Tick/duka_ticks/XAUUSD_post_2025-04_combined.csv
CUT_MS=1743465600000   # 2025-04-01 00:00:00 UTC

if [ ! -f "$TICK_POST" ]; then
    echo "[+] Filtering full corpus to post-Apr-2025..."
    head -1 "$TICK_FULL" > "$TICK_POST"
    awk -F, -v c=$CUT_MS 'NR>1 && $1 >= c' "$TICK_FULL" >> "$TICK_POST"
    wc -l "$TICK_POST"
fi

cd "$REPO/backtest"
# Each engine harness reads $TICK_POST as positional arg-1
g++ -O2 -std=c++17 -I ../include -o duka_sweep duka_sweep.cpp
./duka_sweep "$TICK_POST" --out "../sweep_post_regime_$(date +%Y%m%d_%H%M%S)/"
```

Then re-pick winners using the existing per-engine sweep CSV format and
update each engine's `*BaseParams` struct (CRTP) or constants in
`engine_init.hpp` (legacy assignment layer) following the AsianRange
pattern in this commit.
